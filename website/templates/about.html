About the Project

How does it work?

We use Artificial Intelligence and particularly Machine Learning to predict how many departures and arrivals are going to happen at each station during the next six hours. These numbers are then added the current number of bikes at each station to find those that are going to run empty (zero or one bike), and those that are going to have a 10 or more extra bikes . For each such event we report the time we forecast it's going to happen.

Caveats

  • The model cannot take into account balancing operations that the system operator is doing. Therefore the predictions are approximate.
  • Forecasting is only done for the stations that existed in 2016-2017-2018-2019. The stations added since then are not included in the model.

Who is this for?

The idea of our project is to make a tool for citybike-system operators to make the balancing of bikes much more effective. The added value of improved balancing operations is better user experience, which in turn leads to more cycling, with all the benefits that entails for society. When users learn that they can always find a bike at a station, the system becomes much more usable.

What data did we use?

To train the model we used the bike journey data provided by HSL and the weather observation data from the Finnish Meteorological Institute for the years 2018 and 2019. We decided to not use the data from 2016 and 2017, because the system was not as developed as it is now; or the data from 2020 because it is too much influenced by the coronavirus pandemic. For the real-time predictions we use current station status data from the bike station API of Digitransit and the real-time weather we get from the OpenWeatherMap API. These are then feeded to the model to generate predictions.
